## Phase 8 — Meditations router (`/meditations`)

- [ ] `POST /meditations/create` — authed; intake handler that:
  - Creates `meditations` row (`status: 'pending'`, `visibility` from body, `meditation_array` JSONB snapshot)
  - For each element creates a `jobs_queue` row with the proper `type`, `sequence`, `input_data` (JSON string)
  - `sound` rows are immediately marked `complete` with `file_path` resolved from `prerecorded_audio/`

## Phase 9 — Cascade delete service

- [ ] `src/services/meditations/deleteMeditationCascade.ts` — single function that performs full cascade delete for a meditation:
  1. Delete `jobs_queue` rows for the meditation
  2. Delete `contract_user_meditations` rows for the meditation
  3. Delete the `meditations` row
  4. Delete on disk: all files matching `eleven_labs_audio_files/*/el_{meditationId}_*.mp3`
  5. Delete on disk: `meditation_soundfiles/*/meditation_{meditationId}.mp3`
  6. **Never** touches `prerecorded_audio/`
- [ ] Wrap DB deletes in a transaction; perform filesystem deletes after commit
- [ ] Used by: `DELETE /meditations/:id`, `DELETE /admin/meditations/:id`, `DELETE /admin/queuer/:id`
- [ ] Tests: fixture a meditation with 2 text jobs + 1 sound job + 1 favorite row → run cascade → assert all gone; assert `prerecorded_audio/` untouched
